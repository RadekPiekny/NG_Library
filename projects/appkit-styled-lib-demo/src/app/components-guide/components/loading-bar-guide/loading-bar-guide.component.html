Main

In case of general loading data the best placement is at the top of the application as there is no need to distinguish between desktop and mobile. Other use-cases would be loading at input levels etc.

Saturation

Use different saturation levels only in case of need for a different hiearchy visuals. The lower saturation the lower hiearchy. Prefer global loading bar as handling different loading increase complexity of the code and potential cognitive load.

Speed

If there is some form of slow callback such as data processing you can use a slower speed to indicate the process would take longer. Ideal in that case is to give at least some approximation of the time as well.
