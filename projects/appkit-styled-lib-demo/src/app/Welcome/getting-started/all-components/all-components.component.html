All components

Appkit provides a library of components to meet most of your application's front-end design needs. When used in combination, these components can serve as the building blocks for the layouts and patterns of your application.

Accordion

An element made of a vertically stacked list of items that can be expanded or collapsed.

Avatar

A graphical representation of the user.

Badge

Used as a visual indication or descriptor of status

Breadcrumb

An element used as a navigational aid in user interfaces that tracks the user's location.

Button

Used to initiate a specific action or set of actions.

Checkbox

Used to enable the selection or agreement of one or many items.

Date picker

A date picker enables the user to select a date, date range and/or time.

Dropdown

Displays a standardized list of options to choose from.

Input field

An input field enables users to input text within the context of a form.

Loading

Displays the progress of the application loading or an action within an application.

Modal

Displays in front of and disables main window until user acknowledges by an action.

Pagination

Dividing web content into discrete pages to present in a more understandable format.

Progress

Used to indicate progress within a series of steps.

Radio button

A radio button is a selector used when only one of two or three options should be selected.

Rich text editor

Allows a user to edit and format text.

Slider

Allows users to select a value by moving an indicator - usually vertically.

Tab

Allows multiple documents or panels to be contained in a window.

Table

A table is a data structure used to organize information.

Toggle

A toggle is a control that enables a user to switch between two states.

Tooltip

A brief, informative message that appears when a user interacts with an element.

Upload form

An upload form element allows users to upload external documents into an application.
